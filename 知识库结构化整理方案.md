# 个人知识库结构化整理方案

> Obsidian Vault: JXD&AI | 创建日期: 2026-04-09

---

## 一、项目背景

个人 Obsidian Vault（JXD&AI）已有 118 篇笔记，Phase 1 基础设施（XYZ 三轴标签体系、Templater 模板、Dataview 仪表盘）已搭建完成，但标签覆盖率只有约 20%，双链稀疏。需要批量结构化改造，并建立可持续迭代的处理管线。

---

## 二、核心设计：双层标签涌现体系

### 2.1 第一层：浅层涌现（自底向上）

- **方法**：TF-IDF + HDBSCAN 无监督聚类
- **原理**：从全部笔记的词频-逆文档频率中提取高区分度关键词，让表层主题结构自然涌现
- **产出**：`emergent_tags` —— 数据驱动的语义标签，告诉你"这篇笔记客观上在说什么"
- **关键参数**：出现在 3–15 篇笔记中的词最有价值（太少是孤立概念，太多是背景噪音）

### 2.2 第二层：深层质心牵引（自顶向下）

- **方法**：预定义质心概念 + Embedding 语义距离计算
- **原理**：将用户个人认知框架中的核心锚点概念（如"不隔"等）作为先验注入，反向索引文本，度量每篇笔记与各质心的语义距离
- **产出**：`anchor_tags` —— 质心牵引标签，告诉你"这篇笔记在你的认知框架里意味着什么"
- **类比**：浅层聚类对应工作记忆里的自动关联，深层质心对应长期记忆里沉淀的核心信念和审美框架
- **价值**：能发现跨簇的深层连接——例如"不隔"可能同时吸附古典文学评论和 AI 协作笔记，因为"人机协作的无缝感"和王国维的"不隔"是同一审美倾向

### 2.3 与现有 XYZ 三轴的关系

| 维度 | 名称 | 性质 | 说明 |
|------|------|------|------|
| X 轴 | 主题领域 | 原有，颗粒度粗 | `emergent_tags` 是其细粒度补充 |
| Y 轴 | 认知功能 | 原有，保留不变 | 描述结构性角色 |
| Z 轴 | 拓扑角色 | 原有，保留不变 | 描述结构性角色 |
| 第四维 | `emergent_tags` | **新增** | 数据涌现的语义标签 |
| 第五维 | `anchor_tags` | **新增** | 质心牵引的深层连接标签 |

## 三、存量笔记处理策略

1. **Frontmatter 是纯索引层，不动原文内容。** 在笔记顶部插入 YAML（标签、日期、摘要等），正文完整保留。
2. **双链以增量方式插入。** 在正文合适位置插入 `[[]]` 引用，不破坏原有结构。
3. **不对存量笔记强行套用五段式模板。** 五段式留给新笔记使用。

---

## 四、工程化目标：可复用 Pipeline

### 4.1 核心理念

大模型的角色是**材料理解和分析**，不是每次从头干到尾的劳动力。处理过程必须固定化为流水线，支持持续迭代演进。

### 4.2 Pipeline 流程

```
扫描 vault 中缺 frontmatter/标签的笔记
        │
        ▼
  提取文本做 TF-IDF
        │
        ▼
计算与质心概念的 embedding 距离
        │
        ▼
  调用 LLM API 做语义判断
        │
        ▼
    写回 frontmatter
        │
        ▼
 根据语义相似度建议双链
```

### 4.3 关键设计决策

- 质心概念池为**可迭代的配置文件**，支持随时增删调整
- 新笔记进来**跑一次脚本即可**完成结构化

## 五、实施步骤

### Phase 2a — 分析阶段（当前）

对现有 vault 跑 TF-IDF + 初步聚类，产出分析报告，确定涌现标签候选池和质心概念池。

### Phase 2b — Pipeline 工程化

将处理流程写成可复用的 Python 脚本，放到项目目录。

### Phase 2c — 批量改造

用 pipeline 对存量笔记批量补 frontmatter + 标签 + 双链。

### Phase 2d — 持续迭代

新笔记通过 pipeline 自动处理，质心概念池持续更新。

## 附录：术语速查

- **TF-IDF**：Term Frequency–Inverse Document Frequency，衡量词在特定文档中的重要程度
- **HDBSCAN**：Hierarchical Density-Based Spatial Clustering of Applications with Noise，基于密度的层次聚类算法
- **Embedding**：将文本映射到高维向量空间的表征方法
- **Frontmatter**：Obsidian 笔记顶部的 YAML 元数据区块
- **质心（Centroid）**：此处指认知框架中的核心锚点概念，用于度量语义距离
